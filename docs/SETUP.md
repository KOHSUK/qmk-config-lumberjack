# Lumberjack QMK Firmware - Setup Guide

ZMK (Ortholinix split, 48 keys) からポートした Lumberjack (60 keys, ATmega328p) 用 QMK ファームウェアのセットアップ手順。

---

## 前提条件

| ツール | 用途 |
|--------|------|
| [Nix](https://nixos.org/download/) | 再現可能なビルド環境 |
| [nix-direnv](https://github.com/nix-community/nix-direnv) | ディレクトリ自動環境切り替え |
| USBasp プログラマ | ATmega328p へのフラッシュ |

### Nix インストール（未インストールの場合）

```sh
# 公式インストーラー（Determinate Systems 版推奨）
curl --proto '=https' --tlsv1.2 -sSf -L https://install.determinate.systems/nix | sh -s -- install
```

### nix-direnv セットアップ

```sh
nix profile install nixpkgs#direnv nixpkgs#nix-direnv
# ~/.bashrc または ~/.zshrc に追加:
eval "$(direnv hook bash)"   # bash の場合
eval "$(direnv hook zsh)"    # zsh の場合
```

## セットアップ（クローン後）

```sh
git clone --recursive https://github.com/<user>/lumberjack-keyboard
cd lumberjack-keyboard
direnv allow    # または nix develop
```

`direnv allow` / `nix develop` 時に以下が自動で実行される：
- ビルドツール（avr-gcc, avrdude 等）がシェルに追加される
- `keymaps/kohsuk/` → `qmk/keyboards/peej/lumberjack/keymaps/kohsuk` のシンボリックリンクが作成される

### 確認

```sh
which avr-gcc   # avr-gcc のパスが表示されること
ls qmk/keyboards/peej/lumberjack/keymaps/kohsuk   # symlink 経由でファイルが見えること
```

### 確認

```sh
avr-gcc --version   # avr-gcc x.x.x が表示されること
qmk --version       # QMK CLI バージョン確認
```

## ビルド

```sh
# QMK ホームディレクトリに移動
cd qmk

# ビルド（どちらでも可）
make peej/lumberjack:kohsuk
# または
qmk compile -kb peej/lumberjack -km kohsuk
```

成功すると `peej_lumberjack_kohsuk.hex` が生成される（目安 20–28 KB、32 KB 上限以内）。

## フラッシュ

Lumberjack は USB ブートローダー（USBaspLoader）を搭載しているため、**USB-C ケーブルのみ**でフラッシュできる。

### 1. udev ルール設定（初回のみ・Linux）

root なしで avrdude を実行できるようにする。

```sh
echo 'SUBSYSTEMS=="usb", ATTRS{idVendor}=="16c0", ATTRS{idProduct}=="05dc", TAG+="uaccess"' \
  | sudo tee /etc/udev/rules.d/50-usbasp.rules

sudo udevadm control --reload-rules
sudo udevadm trigger
```

設定後、**USB-C を一度抜いて再接続**する。

### 2. ブートローダーへの入り方

1. USB-C を接続した状態で
2. **BOOT ボタン（SW2）を押したまま保持**
3. **RESET ボタン（SW1）を押して離す**
4. **BOOT ボタン（SW2）を離す**

キーボードとして認識されなくなり、USBasp デバイスとして見えればブートローダーモード。

> 注意：BOOT ボタンは Column 3 と共有しているため、押下中にそのカラムのキーも反応するが正常な動作。

### 3. フラッシュ

```sh
avrdude -c usbasp -p atmega328p -U flash:w:peej_lumberjack_kohsuk.hex:i
```

### 4. 再起動

フラッシュ完了後、**RESET ボタン（SW1）を単独で押す**と新しいファームウェアで起動する。

## キーマップリファレンス

### レイヤー構成

| レイヤー | 名前 | 起動条件 |
|---------|------|---------|
| 0 | BASE | デフォルト |
| 1 | FN1 | LT1 スペース（右）ホールド |
| 2 | FN2 | LT2 スペース（左）ホールド |
| 3 | FN3 | LT3 Enter ホールド |
| 4 | FN4 | LT4 Backspace ホールド |
| 5 | FN5 | MO(FN5) ホールド |
| 6 | FN6 | LT6 スペース（最右）ホールド |
| 7 | FN7 | LT7 = ホールド |

### Row 対応（ZMK → Lumberjack）

| ZMK | Lumberjack | 内容 |
|-----|-----------|------|
| なし | Row 0 | 数字行: ESC 1-0 BSPC |
| Row 0 | Row 1 | = Q..T / Y..P - |
| Row 1 | Row 2 | TAB A..G / H..SCLN QUOT |
| Row 2 | Row 3 | LSFT Z..B / N../ RSFT |
| Row 3 (14 keys) | Row 4 (12 keys) | 親指クラスタ（BT キー省略） |

## トラブルシューティング

### ビルドエラー: `avr-gcc not found`

```sh
# シェル環境が有効になっているか確認
which avr-gcc
# 表示されない場合
nix develop    # または direnv allow
```

### avrdude: `could not find USB device` / `Permission denied`

1. udev ルールが設定されているか確認（「udev ルール設定」セクション参照）
2. USB-C を一度抜いて再接続したか確認
3. ブートローダーモードに入っているか確認（BOOT + RESET 手順）

### avrdude: `initialization failed, rc=-1`

- ブートローダーモードに入れていない可能性がある → BOOT + RESET の手順を再試行
- USB-C ケーブルがデータ通信対応か確認（充電専用ケーブルは不可）

### LT キーが誤作動する（スペース連打でレイヤーが起動する）

`config.h` で `PERMISSIVE_HOLD` が無効になっていることを確認。
`TAPPING_TERM` (デフォルト 200ms) を調整することで改善できる場合がある。

### `.hex` ファイルが 32 KB を超える

- `mousekey` や `extrakey` 機能を `keyboard.json` で無効化
- 不要なレイヤーを削除
